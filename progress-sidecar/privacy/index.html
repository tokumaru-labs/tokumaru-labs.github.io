日本語（正本）

1. 収集・保存する情報

Progress Sidecar は、次の情報を ユーザーがこの拡張機能に入力・操作した範囲でのみ 扱います。

  • タスク情報（タスク名、完了量、合計量、単位、1単位あたりの予想時間などユーザーが入力した内容）
  • 進捗の更新
  • 表示設定（表示言語などの設定）
  • 削除時に自動生成されるローカルの復元用スナップショット（recovery snapshot）

上記以外の情報は取得しません。氏名・メールアドレス・位置情報・閲覧履歴・アカウント情報などの個人識別情報を要求・収集することはありません。

2. 利用目的

保存した情報は、拡張機能の本来の機能（総量・完了・残り・推定残り時間の表示、タスクの追加・編集・進捗更新、バックアップの書き出し／読み込み、削除タスクの復元）を提供する目的にのみ利用します。

3. 保存場所

すべての情報は、お使いの端末の Chrome が管理する chrome.storage.local（この端末のローカル領域）にのみ保存されます。

  • chrome.storage.sync は使用しません（他端末へ同期されません）。
  • 外部サーバーやクラウドには保存しません。

4. 外部送信

Progress Sidecar は、保存した情報を外部サーバーへ送信しません。ネットワーク通信を行う機能を持ちません。

5. 第三者提供

保存した情報を第三者に提供・販売・共有することはありません。開発者を含め、いかなる人間もユーザーの入力データを閲覧しません。

6. 広告・分析

広告配信、アクセス解析（analytics）、トラッキング、テレメトリは一切行いません。これらのための外部スクリプトやSDKも含みません。

7. データの書き出し（Export）と読み込み（Import）

  • Export：ユーザーが操作したときにのみ、保存内容を JSON ファイルとしてこの端末にダウンロードします。送信先は端末のローカルファイルのみです。
  • Import：ユーザーが選択したローカルの JSON ファイルのみを読み込みます。読み込み前に内容を検証し、既定では既存データへの追加・更新（Merge）として反映します。

8. データの削除方法

  • 各タスクは、拡張機能内の削除操作で削除できます。
  • Chrome の設定から拡張機能のサイトデータを消去することでも削除できます。
  • 拡張機能を Chrome から削除すると、chrome.storage.local に保存された情報も Chrome によって削除されます。

書き出した JSON ファイルは端末上のユーザー管理下のファイルであり、必要に応じてユーザー自身が削除してください。

9. 拡張機能を削除したときの扱い

拡張機能を削除すると、拡張機能のローカルストレージ内容は Chrome によって消去されます。復元用スナップショットは端末外に保管されないため、削除前に必要なデータは Export で保存してください。

10. セキュリティ

データはネットワークを経由せず、この端末の Chrome ローカルストレージ内に留まります。拡張機能が要求する権限は sidePanel と storage のみで、ホスト権限やページへのスクリプト挿入は行いません。

11. ポリシーの変更

本ポリシーを変更する場合は、本ページを更新し、バージョンと最終更新日を改訂します。

12. 問い合わせ先

English (reference translation)

1. Information stored

Progress Sidecar handles only the information you enter or act on inside the extension:

  • Task data you enter (title, completed amount, total amount, unit, estimated minutes per unit, and similar fields)
  • Progress updates
  • Display settings (such as interface language)
  • Local recovery snapshots created automatically when you delete a task

Nothing else is collected. The extension does not request or collect personally identifying information such as your name, email address, location, browsing history, or account details.

2. Purpose of use

Stored information is used only to provide the extension's core features: showing total, completed, remaining, and estimated remaining time; adding, editing, and updating tasks; exporting and importing backups; and restoring deleted tasks.

3. Where data is stored

All information is stored only in chrome.storage.local managed by Chrome on your device.

  • chrome.storage.sync is not used (data is not synced to other devices).
  • Nothing is stored on external servers or in the cloud.

4. No external transmission

Progress Sidecar does not transmit stored information to any external server. It contains no networking feature.

5. No third-party sharing

Stored information is never provided, sold, or shared with third parties. No person, including the developer, can view your entered data.

6. No advertising or analytics

The extension performs no advertising, analytics, tracking, or telemetry, and bundles no external scripts or SDKs for those purposes.

7. Export and import

  • Export: only when you choose to, the current data is downloaded to your device as a JSON file. The destination is a local file on your device only.
  • Import: only a local JSON file you select is read. Its contents are validated before anything is written, and by default it is applied as an additive Merge into existing data.

8. How to delete your data

  • Delete individual tasks with the delete action inside the extension.
  • Clear the extension's site data from Chrome settings.
  • Removing the extension from Chrome causes Chrome to delete the data stored in chrome.storage.local.

Any JSON file you exported is a file under your control on your device; delete it yourself if desired.

9. When you remove the extension

When you remove the extension, Chrome clears its local storage contents. Recovery snapshots are not stored outside your device, so export any data you want to keep before removing it.

10. Security

Data never travels over the network and stays inside Chrome's local storage on this device. The extension requests only the sidePanel and storage permissions and does not use host permissions or inject scripts into web pages.

11. Changes to this policy

If this policy changes, this page will be updated, and the version and last-updated date will be revised.
